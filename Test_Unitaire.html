    Comment tester "unitairement" le code du Controller ?

Bonnes pratiques

  • TDD
    • Première étape : écrire le test et l'exécuter (rouge)
    • Deuxième étape : écrire la fonction pour que le test passe (vert)
    • Troisième étape : refactoring
  • Un test par fonction et par comportement
  • Pour les fonctions plus complexes, tester en pair programming
  • Vérifier la couverture des tests : phpunit --coverage-html [directory]

Contraintes

Dans le cas des tests avec Laravel,

  • Il n'y a plus le contexte de l'application
  • L'autoload n'est pas géré.
